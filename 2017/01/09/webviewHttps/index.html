WebView加载https ssl认证问题

发表于 2017-01-09   |   分类于 Https   |  

  1. 错误信息

        primary error: 5 certificate: Issued to: CN=h5.immocha.com;Issued by:CN=Symantec Basic DV SSL CA - G1,OU=Domain Validated SSL,OU=Symantec Trust Network,O=SymantecCorporation,C=US;
onURL:https://h5.immocha.com/mbox/vip/showVip.html ------ ERROR

  2. 解决办法

调用SslErrorHandler 中的proceed()方法
作用就是继续让请求进行认证，返回解雇

@Override
public void onReceivedSslError(WebView view, SslErrorHandler handler, SslError error) {
        //在进行https认证时有时会报错
        Log.e("error" , "==" + error);
        handler.proceed();
        super.onReceivedSslError(view, handler, error);
    }

查看WebViewClient 源码，发现onReceivedSslError，默认调用request.cancel();
而onReceivedClientCertRequest方法作用：Notify the host application to handle a SSL client certificate request. The host application is responsible for showing the UI if desired and providing the keys. There are three ways to respond: proceed(), cancel() or ignore(). Webview stores the response in memory (for the life of the application) if proceed() or cancel() is called and does not call onReceivedClientCertRequest() again for the same host and port pair. 默认调用cancle();
