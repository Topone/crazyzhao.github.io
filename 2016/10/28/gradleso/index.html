Androidstudio gradle配置动态库.so

发表于 2016-10-28   |  

主要讲AndroidStudio中怎么配置Gradle，让Gradle在打包apk时只生成我们所需要的某些abi文件夹

最近项目打包apk后，出现部分手机初始化失败,类似如下error：

java.lang.UnsatisfiedLinkError:nativeLibraryDirectories=[/data/app/com.xxx.xxxxxxx] couldn’t find “libxxxx.so” 
java.lang.UnsatisfiedLinkError: dlopen failed: “libs/armeabi-v7a/libxxxxxx.so” xxxxxxxxxxxxxxxxxxxxxxx 
java.lang.UnsatisfiedLinkError: xxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

由于是引入Fresco之后出现的异常，解压apk后看了下libs目录，发现有armeabi、armeabi-v7a、arm64-v8a、x86、x86_64五种ABI，主要原因由于手机cpu的差异性，导致加载支持abi文件夹的不同，某些第三方库在arm64-v8a包下没有生成.so,所以造成初始化时无法加载，导致崩溃。

解决方案：

一. 修改build.gradle文件配置

android {
    defaultConfig {
        ndk {
            //生成哪些abi文件
            abiFilters 'armeabi' ,'armeabi-v7a'
            }
        }
    }

然后在project的根目录(和local.properties同级)修改gradle.properties，文件里面加入下面这行：

android.useDeprecatedNdk=true

二.由于有多种解决方案，包括packagingOptions设置:

android {
    packagingOptions {
        //有效
        exclude 'lib/arm64-v8a/libbitmaps.so'
        //无效
        exclude 'lib/arm64-v8a'
        }

只是针对单个so文件做排除，并不支持文件夹。所以还是推荐使用第一种方案
